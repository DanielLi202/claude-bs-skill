# r2 — cycle-028 process review (B-030)

Summary: r1 found two real process escapes. F1 is a proof-validity escape: the Grade accepted an active-path containment proof without proving the proof fails closed when the containment primitive is inactive. F2 is a trust-boundary escape: the Grade trusted `sandbox-exec` as a base OS primitive but never required evidence that the trusted binary was invoked by absolute path rather than PATH. Deterministic lint can catch the missing evidence shape; it cannot prove the Rust source actually implements the asserted behavior.

## F1 — P0 — containment proof passes when sandbox is inactive

Owning gate: medium/high code adversarial Grade: `risk_surface` / `adversarial_acceptance` + Grade `adversarial_checks` / `negative_regression_tests`, backed by `grade_verify` and `grade_lint`.

Why it missed:
- `outcome.md` adv1/a3 required an obfuscated forbidden-root read to run under sandbox and prove secret absence.
- `outcome.md` adv5 covered sandbox-unavailable fallback, but only asserted normal fallback still runs and a literal textually-forbidden command is rejected.
- Round-1 Grade passed because `cargo nextest` ran on a host where `sandbox-exec` was active; the inactive branch was not exercised by the proof test.
- Existing `grade_lint.py` correctly checked adversarial block shape, surface coverage, subprocess lifecycle facets, symlink/canonical containment, and trust inventory accounting. It did not require “the P0 containment proof itself fails/refuses when the primitive is unavailable.”
- A mechanical lint cannot inspect `if !probe.active { return; }`; it can only require explicit fail-closed unavailable-primitive evidence in the Grade text.

Process conclusion: add a deterministic evidence-shape rule for unavailable containment primitives, plus a human semantic review requirement for source/test proof fail-closedness.

## F2 — P1 — trusted binary discovered but invoked through PATH

Owning gate: adversarial Grade trust review: `trust_surface_inventory` for `external_subprocess` / `file_modes`, plus Grade semantic code-path review. `grade_lint` can only enforce evidence text, not source-code invocation.

Why it missed:
- The capsule said `sandbox-exec` is a base macOS binary and the wrapper enforces containment, but no acceptance or trust row required absolute-path invocation or PATH-precedence poisoning.
- Grade evidence discussed canonical absolute forbidden-root paths, not the absolute path of the `sandbox-exec` binary.
- `grade_verify` did not run a fake-`sandbox-exec`-first-in-PATH test.
- `grade_lint.py` reads outcome/Grade text only; it cannot see `Command::new("sandbox-exec")`.

Process conclusion: add a deterministic “trusted containment binary must have absolute-path / non-PATH evidence” lint. Actual confirmation still needs code-aware review because prose can assert absolute invocation falsely.
