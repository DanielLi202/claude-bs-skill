Read-only process review. Grade 0 already failed; F7-F11 are not “clean Grade passed” escapes. They are review-completeness/fix-directive escapes: the failing Grade did not discover these extra P1s, so the fix round could not target them.

F7 — capsule schema guard escape:
- Should-have gate: Grade rubric, with possible `grade_lint` evidence-completeness support.
- Why it missed: Grade had `adv1` as unverified and blocking, but only at example level: malformed arrays / missing type / missing fields. It did not audit `capsuleModel.ts` as a schema-v1.2 guard against top-level required fields, controlled tags, high-risk action enum, source-specific grounding locators, and non-empty `supports[]`. Current `grade_lint.py` checks accounting/status and Shape-agent outcome front matter, not frontend parser fixture validity.

F8 — tags parsed but not rendered:
- Should-have gate: Shape + Grade UI/prototype compliance.
- Why it missed: the locked prototype explicitly says non-empty `outcome.tags[]` surfaces as chips, but the shaped “What to build” did not carry that as an explicit acceptance, and Grade did not inspect the prototype-required tag row. Existing frontend lint source-affordance checks cover prior Inspector/Shelf/Pellet sources, not this Outcome prototype tag affordance.

F9 — stale RunDetail overrides fresher snapshot/revision:
- Should-have gate: Grade semantic state-authority review.
- Why it missed: Grade accepted broad “no fabricated stage” / conflict-retry framing but did not inspect the actual dataflow where `detail` overrides `run`, detail fetch is not revision-keyed, and `currentRevision` prefers stale detail. This is not a safe regex lint target unless a required state-authority evidence matrix is added; it needs human semantic review.

F10 — re-shape/reject not pessimistic:
- Should-have gate: Grade rubric plus frontend pessimistic-command lint.
- Why it missed: current v1.4.20 lint checks pending-dismissal facets only for pass claims and did not require a command-by-command DA-28/29 matrix. Grade focused on Execute and crash-blocked tests; it did not separately require pending/scoped disable/inline error/catch for `re-shape mode=continue` and assumption reject.

F11 — 409 then refetch failure strands pending:
- Should-have gate: adversarial concurrency Grade.
- Why it missed: `adv3` covered one-2xx/one-409, refetch, and retry with fresh revision, but not the negative branch where the refetch itself fails. Current `grade_lint.py` only enforces `evidence_kind: concurrency_test`; it does not require failure injection after awaited conflict refetches.

CONV — two wasted fix-round turns:
- Should-have gate: Conduct runtime + driver.
- Why it missed: `outcome.md` fix corrections start at line 303, both vendors read only lines 1-240, and `BS_OUTCOME_READ` proved only path/sha. `codex_fix_driver.py` is just `codex_driver.main`, so the fix-round objective stayed generic. `semantic_validated=true` meant “some workspace delta exists,” not correction uptake. `conduct.sh` only runs the existing alignment gate on `rc == 0`; both attempts were `rc == 6`. Even if it had run, loci-touch alignment is insufficient: touching `outcomeStore.ts` or `capsuleModel.ts` does not prove any folded correction landed.

GOAL — repeated `goal_active_after_turn`:
- Should-have gate: driver success oracle.
- Why it missed: the agent completed work but left the goal status active; contract correctly treats active as non-success. Changing this oracle or auto-completing goals is not deterministic safety work; at most add an opt-in completion nudge.
