## Escaped r1 findings

### F1 — probe subprocesses hang / child-spawn escape

**Gate that should have caught it:** Shape/Grade adversarial subprocess lifecycle gate: `B018-A9`, `B018-ADV-SUBPROC-1`, `B018-ADV-PROBE-FAILCLOSED-1`, then `runtime/grade_lint.py::validate_adv`.

**Why it escaped:** `grade_lint.py` only checked that `adversarial_checks` existed, had `evidence_kind: subprocess_lifecycle_test`, and referenced the shaped adversarial IDs. It did not require lifecycle facets for *every subprocess class*. The Grade evidence asserted handoff/probe safety, but the actual probe helpers `codex --version`, `codex login status`, `claude --version`, `claude auth status`, and Claude ping used raw `Command::output().await` without timeout/process-group/reap.

**Minimal catch:** Add a subprocess lifecycle facet rule: if a P0/P1 acceptance/adversarial check mentions external subprocess, probe, version/auth/ping, timeout/cancel, or process-group reaping, the Grade row must cite evidence covering timeout + process group + wait/reap for each subprocess surface, including capture/probe commands.

### F2 — timeout signals but does not reap child/tasks

**Gate that should have caught it:** Same subprocess lifecycle gate, especially `B018-A9` and `B018-ADV-SUBPROC-1`.

**Why it escaped:** The Grade row equated `terminate_process_group(pid)` with “reaped.” `grade_lint.py` did not require evidence of `child.wait()` after timeout, nor joining stdout/stderr tasks. The actual `run_vendor_stream()` timeout branch returns immediately after signaling.

**Minimal catch:** In the same lifecycle facet rule, require timeout/cancel evidence to include actual reap semantics: child wait/try_wait after signal and stdout/stderr task join or equivalent cleanup proof.

### F3 — Codex timeout skips `goal/clear` + `thread/archive`

**Gate that should have caught it:** `spec_compliance_matrix` / `negative_regression_tests` for `B018-A6` cleanup-on-every-exit-path.

**Why it escaped:** `B018-A6` Grade evidence covered the 5s inferred-completion path, not the outer timeout path. `grade_lint.py::validate_property_obligations` only has hardcoded property facets for path/root containment and HTTP request-targets; it has no RPC cleanup-on-timeout obligation.

**Minimal catch:** Add a cleanup property rule: if a P0/P1 acceptance says cleanup/clear/archive happens on every exit path, the negative row must include timeout/error-path evidence proving both `thread/goal/clear` and `thread/archive`.

### F4 — Claude `system/init` and `result` goal snapshots dropped

**Gate that should have caught it:** `spec_compliance_matrix` for `B018-A7`/`B018-A4`, plus exit-condition `B018-A8`.

**Why it escaped:** The matrix checked aggregate “≥1 goal_snapshot” and accepted a fake Claude stream that included `post_turn_summary`. It did not require each spec-required source event to be emitted. The implementation created goal snapshot values for `system/init` and `result` but dropped them because `ingest_line()` appends only the single returned event.

**Minimal catch:** Add an event-emission source obligation: when an acceptance names required vendor source events and normalized outputs, Grade must provide a per-source mapping, e.g. `system/init -> goal_snapshot`, `result -> goal_snapshot`, with fixture/test evidence not masked by alternate sources.

### F5 — secret redaction misses JSON/header/quoted token shapes

**Gate that should have caught it:** `secret_leakage_audit` and `B018-ADV-SECRET-1`.

**Why it escaped:** `grade_lint.py` accepted `cleartext_secret_probe: pass` plus a generic evidence ref. The Grade evidence only covered `token=sk-secret-test` and AdapterError strings. It did not require JSON/colon, quoted, or `Authorization: Bearer` probes, nor proof that `vendor_stderr.txt` was clean.

**Minimal catch:** Tighten `secret_leakage_audit` validation to require structured or text-detected probe shapes for auth/secret surfaces: `token=`, JSON/quoted API key/token, and `Authorization: Bearer`, across Debug/Display, tracing/logs, events, and evidence files.

### F6 — login-status parsing brittle

**Gate that should have caught it:** `negative_regression_tests` for `B018-A5` / `B018-A3` distinct `LoginRequired`.

**Why it escaped:** The negative test used “not logged in” prose and a secret leak assertion. It did not require format-tolerant status parsing or JSON variants like `{"loggedIn": false}` with whitespace/case differences. `grade_lint.py` has no auth-status parsing obligation.

**Minimal catch:** Add an auth-status parsing obligation: if a P0/P1 acceptance claims `codex login status` / `claude auth status` maps to `LoginRequired`/`NotAuthenticated`, negative evidence must cover JSON-parsed/whitespace-tolerant status output, not only prose substring matching.

## Separate process escape: merged PR but Step 10 never ran

**What happened:** `auto_merge_gate.yaml` says `decision: merge` for PR #25. `main` contains the PR merge commit. But backlog still has `B-018 status: in_progress`, no `dogfood-log.md` cycle-018 entry exists, `AGENTS.md` still points to B-018, and the cycle branch/worktree remain. `step_events.jsonl` ends at `step_7 started`.

**Why workflow allowed it:** `/bs` has Step-10 close rules, but no durable recovery guard for “PR already merged, close not run.” On the next invocation, `/bs` preconditions only reject `in_progress`; they do not diagnose “merged PR needs close.” `/bs resume` is strict about an open started event, but lacks a special verified recovery branch for open `step_7` with a merged PR.

**Minimal guard:** Add a read-only close-gap doctor in preflight/status/doctor, and a `/bs resume` recovery path:

- Detect exactly one `in_progress` task whose latest cycle has `auto_merge_gate.decision: merge`, a PR URL/number, and that PR is already merged into `origin/main`.
- Refuse new-cycle start with `recovery_required: merged_pr_needs_step10_close`.
- `/bs resume` may append `step_7 completed` only after verifying the PR merge SHA/main containment, then proceed directly to Step 10 atomic close.
