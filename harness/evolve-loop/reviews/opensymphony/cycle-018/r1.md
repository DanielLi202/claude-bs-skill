## Independent review result

Verdict: **FAIL**. I found multiple P1 escapes beyond the workflow Grade, mostly around subprocess lifecycle, timeout cleanup, Claude goal-snapshot normalization, and secret redaction.

### Key escaped findings

1. **Probe subprocesses are not fail-closed under hang/child-spawn conditions.**  
   `codex --version`, `codex login status`, `claude --version`, `claude auth status`, and Claude ping use raw `Command::output().await` with no timeout, no `.process_group(0)`, and no reap logic. This contradicts B018-A5/A7/A9’s fail-closed probe and process lifecycle requirements.

2. **Claude/generic timeout path does not reap the child.**  
   `run_vendor_stream()` returns immediately after `terminate_process_group(pid)` on timeout, without `child.wait().await` or joining stdout/stderr tasks. This can leave zombies/tasks and does not satisfy “reap whole process group on every exit path.”

3. **Codex timeout skips `goal/clear` + `thread/archive`.**  
   Cleanup is inside the future wrapped by `time::timeout`; if the timeout fires, the code kills the app-server and returns `VendorTimeout` without issuing clear/archive. B018-A6 explicitly requires cleanup on every exit path.

4. **Claude `system/init` and `result` goal snapshots are silently dropped.**  
   `normalize_claude()` constructs `GoalSnapshotPayload`s for init/result but ignores the returned event; `ingest_line()` only appends the single event returned by `normalize_claude()`. Fake tests pass only because they include `post_turn_summary`, masking the missing required signals.

5. **Secret redaction is format-fragile and leaks common token shapes.**  
   Redaction only handles whitespace tokens like `token=`/`api_key=`/`sk-`; JSON/colon/header forms such as `{"api_key":"sk-..."}` or `Authorization: Bearer ...` survive into `vendor_stderr.txt`. The Grade secret test covered only the easiest `token=...` form.

6. **Codex login-status detection is brittle.**  
   It checks literal `"loggedin":false` without tolerating whitespace or real JSON parsing. A valid `{"loggedIn": false}` response maps away from `LoginRequired`, violating the distinct-remediation requirement.
